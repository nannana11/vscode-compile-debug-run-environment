# 在windows操作系统配置vscode的c和cpp编程环境时在配置简单的编译、调试、运行阶段所需要的文件
## 这个仓库为谁提供服务？
为想要在vscode上使用到**和devc++类似的编译调试运行操作**的人

在windows操作系统配置vscode的c、cpp编程环境分为以下几个步骤：

1，下载并安装vscode 

2，下载拓展包，如c、cpp拓展包（对于中国用户如有需要可下载中文拓展包） 

3，下载MinGW,并将其中的bin文件的路径添加到系统环境变量中的path 

4，配置编译、调试和运行的环境。   

这个仓库是帮助【已经完成了前三个步骤】去完成【第四个步骤】的人准备的，即对于已经安装好了vscode并且将gcc、gbd配置到了path里面的人，这个仓库里面的文件将帮助你配置好编译、调试和运行的环境

【注意！】本仓库内文件只支持【单文件】编译运行！

## 这个仓库中内容的特点
- 【种类丰富，适合不同人不同机器的需要】想要使用c89标准进行编译或c99标准进行编译，这里都有对应的版本
- 【操作简单，直接复制即可使用，对于想要自定人的人也附有修改说明】
- 【完成后与devc++的编译和运行基本相同，适合新手使用】会生成与devc++类似的“黑色窗口”(即powershell)，运行和调试都在窗口进行，窗口在程序结束后不会立即消失

## 内容说明
1，在vscode进行编译运行更多情况下其实是直接在它自己的终端上操作的，不会借用外部的终端，比如windows的cmd或者powershell，但是鉴于入门的人都由于刚从devc++等转来，更熟悉外部的终端（windows的窗口），故本仓库内的.json都会生成外部的窗口，且是powershell而非cmd

2，windows的窗口在程序完成后会立即自动关闭，这样会导致我们有时候还没看到程序运行结果是什么，窗口就关闭了，为此tasks.json里面增加了一个任务，用于保证窗口永远不主动关闭，也因此我们不能直接使用vscode右上角的“运行”，不然窗口仍然会自动关闭，而要ctrl+shift+p选择"任务运行任务"选择“run the program,keep the window”，这样非常麻烦，所以我们选择了额外添加快捷键

3，调试直接使用vscode右上角的“调试”即可

## 操作说明
主要分为两个步骤：

1，添加tasks.json文件和launch.json文件以配置编译、调试、运行环境
- 在仓库中选择一个描述与你的需求相符合的文件夹
- 打开这个文件夹，下载其中的.vscode文件夹到你的电脑
- 将这个.vscode文件夹和你需要进行编译运行的.c或者.cpp文件放在同一文件夹内，且保证它们处于同一级，即需要这样的文件结构：

- file
- - .vscode
- - - tasks.json
- - -launch.json
- - target_file.c

2,添加编译运行的快捷方式：此仓库内所有文件默认为：alt+f9
- 在vscode界面内，先按下ctrl+k，然后按下ctrl+s，进入快捷键设置页面
- 在右上角找到一个图标，名字是【打开键盘快捷方式（JSON）】，点击后自动生成一个keybinding.json文件
- 在本仓库内找到名为【keybinding】的文件夹，其中有个.json文件，将其中内容复制粘贴到keybinding.json文件中，保存即可（注意，这个keybinding.json文件在生成时就会保存在这个地址"C:\Users\lenovo\AppData\Roaming\Code\User\keybindings.json"）

完成以上两个步骤后，就可以进行编译、调试和运行了！！

直接编译运行（不调试直接运行）方法，编写完文件后直接按下alt+f9快捷键

调试方法，在.c文件左侧设置了红色断点后，点击右上角【调试】按钮，即可开始调试
